## 尝试安装 self-improving-agent skill

### 遇到的问题

1. **Slug 格式错误**
   - 问题：使用 `pskoett/self-improving-agent` 作为 slug，但实际这个 skill 的所有者是 `ivangdavila`
   - 教训：在安装前应该先用 `npx clawhub search` 确认正确的 skill 名称和所有者

2. **Rate Limit Exceeded（速率限制）**
   - 问题：clawhub API 调用频率过高被限制
   - 教训：
     - 避免在短时间内多次调用 clawhub 命令
     - 如果遇到速率限制，等待一段时间再试
     - 可以先用 `inspect` 命令查看信息，而不是直接尝试安装

3. **GitHub 仓库 404 错误**
   - 问题：直接猜测 GitHub 仓库路径失败
   - 教训：
     - 不要假设仓库路径格式，先确认正确的仓库地址
     - 可以通过 clawhub 的 `inspect` 命令获取更多信息，而不是盲目尝试

### 以后的改进策略

1. **安装 skill 前的检查清单**：
   - [ ] 先用 `npx clawhub search <关键词>` 搜索
   - [ ] 确认正确的 skill slug 和所有者
   - [ ] 用 `npx clawhub inspect <slug>` 查看详细信息
   - [ ] 确认无误后再安装

2. **GitHub 访问策略**：
   - [ ] 先从可靠来源获取仓库地址，而不是猜测
   - [ ] 如果需要访问 GitHub，考虑使用 GitHub API 而不是直接猜测路径

3. **错误处理**：
   - [ ] 遇到问题时，先记录问题和可能的原因
   - [ ] 不要盲目重试，先思考其他方法
   - [ ] 把每次问题都当作学习机会

### 学到的东西

- 即使是看起来简单的任务，也可能遇到各种问题
- 事先做足调查可以避免很多麻烦
- 记录问题和解决方案是自我改进的关键

✅ **pnpm 镜像源** - 已配置为淘宝镜像
- 地址：https://registry.npmmirror.com
- 配置命令：`pnpm config set registry https://registry.npmmirror.com`

✅ **nvm Node.js 镜像源** - 已配置为淘宝镜像
- 地址：https://npmmirror.com/mirrors/node/
- 配置：`export NVM_NODEJS_ORG_MIRROR=https://npmmirror.com/mirrors/node/`

### 环境配置脚本

✅ **创建了 env-setup.sh**
- 位置：/root/.openclaw/workspace/env-setup.sh
- 功能：一键加载 nvm、pnpm 和配置所有镜像源

### 使用建议

- 日常使用 pnpm 替代 npm，速度更快

- 新会话中运行 `source env-setup.sh` 快速配置环境

## 迁移到 nvm 管理 Node.js (2026-03-06)

老浪要求卸载系统 npm，统一使用 nvm 的 LTS 版本，并重新安装 Claude Code。

### 执行步骤

1. **卸载系统 Node.js**
   - 系统 Node.js 是通过 nodesource deb 包安装的 (v22.22.0)
   - 卸载命令：`apt-get remove -y nodejs`

2. **通过 nvm 安装 Node.js LTS**
   - 配置国内镜像源：`export NVM_NODEJS_ORG_MIRROR=https://npmmirror.com/mirrors/node/`
   - 安装命令：`nvm install --lts`
   - 安装的版本：Node.js v24.14.0 (LTS)，npm v11.9.0
   - 自动设置为默认版本：`default -> lts/* (-> v24.14.0 *)`

3. **配置 npm 镜像源**
   - 设置为淘宝镜像：`npm config set registry https://registry.npmmirror.com`

4. **重新安装 Claude Code**
   - 安装命令：`npm install -g @anthropic-ai/claude-code`
   - 安装速度：只用了 2 秒！（镜像源效果显著）
   - 版本：Claude Code 2.1.70

### 当前环境状态

✅ **Node.js**: v24.14.0 (LTS) - 通过 nvm 管理
✅ **npm**: v11.9.0 - 随 Node.js 一起安装
✅ **npm 镜像源**: https://registry.npmmirror.com
✅ **Claude Code**: 2.1.70 - 全局安装
✅ **nvm**: 0.40.1 - Node.js 版本管理器

### 使用说明

- 新会话中需要加载 nvm：
  ```bash
  export NVM_DIR="$HOME/.nvm"
  [ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"
  ```
- 或者直接运行 `source env-setup.sh` 一键配置环境

### ⚠️ 重要提醒

**以后所有 npm 命令都用 pnpm 替代！**
- `npm install` → `pnpm install`
- `npm install -g` → `pnpm add -g`
- `npm run` → `pnpm`
- 等等...

pnpm 更快、更省磁盘空间，优先使用！
